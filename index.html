Project Description

We were first introduced to Angular in SE 396 (assignment six). In this class we used Angular to create a corporation finder. This included a search feature and the ability to drill down into the corportation details. While it was nice to start working with a new language, this assignment only provided us with a taste of what Angular could do. For this class I would like to study Angular in more depth. My goal will be to create a website for an anual memorial run. I plan to use the MEAN stack to create this application.

Application Details:

  • Provide information about the cause
  • Display pictures of past runs and events
  • Provide a contact form for users
  • Provide a sponsor form for potential sponsors
  • Interactive course information for the upcoming run
  • Handle registration for upcoming run and events

This week I continued the Udemy course. I learned about NoSQL databases, specifically MongoDB which is a document store. Having only dealt with relational databases in school, this is a tough concept to grasp (1 hour). In order to work with MongoDB I needed to first install it on my Mac. I ran into a few issues since this Udemy class assumes you already have MongoDB installed and simply sends you to mongodb.org for a download if you have not. Once I was able to get the MongoDB installed and running I learned how to create a new database, new collection and then documents, followed by the basic CRUD operations for documents and collections. We then covered how to export the database as BSON or JSON and how to zip the data. We also covered how to create new databases and collections from a JSON file (3 hours). Finally, I updated the existing project to use a database with MongoDB instead of the original JSON file. I am now able to get all hotels, one hotel and add a hotel to the database (2 hours). Next week I will start building out the REST API and dive into Mongoose.
Week Five - Sunday, February 05, 2017
This week's Udemy classes covered how to use Mongoose and how to define a Mongoose schema. This included schema types, basic validation, setting default values and being able to do this all in one place with JavaScript (2 hours). I then learned how to define and include subdocuments or schemas into the main schema, store geo-location coordinates and create a spherical index for geo-spatial data with 2D indexes for searching (3 hours). I used Mongoose queries for GET routes of the API to get multiple hotels, a subset of hotels and a single hotel. Then drilled further down to get multiple and then individual sub-documents (2 hours). I then used Mongoose to find hotels within a certain distance of other hotels. More excitingly, I learned how to add to the parent and sub-documents to add new reviews to existing hotels. I finished the week by completing the API with the remaining CRUD methods, update and delete (3 hours). Next week I will start learning about single page applications using AngularJs that will use the API I have just completed building.
